Detailed Description

Perform a logical operation on an image. 

Logical operations on one or two images.
NOTE: The output of this filter is an image with pixels of values 0 and 1.
An appropriate scaling must be performed either manually (with pxrescaleintensityimagefilter)
or with the application used to view the image.

In the case of a vector image, this is a componentwise logical operator.
Usage:
pxlogicalimageoperator
  -in      inputFilename1 [inputFilename2]
  [-out]   outputFilename, default in1 + <ops> + in2 + .mhd
  -ops     LogicalOperator of the following form:
             [!]( ([!] A) [{&,|,^} ([!] B])] )
           notation:
             [NOT_][NOT][{AND,OR,XOR}[NOT]]
           notation examples:
             ANDNOT = A & (!B)
             NOTAND = (!A) & B
             NOTANDNOT = (!A) & (!B)
             NOT_NOTANDNOT = !( (!A) & (!B) )
             NOT_AND = !(A & B)
             OR = A | B
             XOR = A ^ B
             NOT = !A 
             NOT_NOT = A
           Internally this expression is simplified.
  [-z]     compression flag; if provided, the output image is compressed
  [-arg]   argument, necessary for some ops
  [-dim]   dimension, default: automatically determined from inputimage1
  [-pt]    pixelType, default: automatically determined from inputimage1
Supported: 2D, 3D, (unsigned) short, (unsigned) char.
